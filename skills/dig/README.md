# dig

プランや設計ドキュメントに潜む暗黙の前提・未検討リスク・未決定事項を、構造化質問の反復インタビューで掘り起こすためのスキルです。明確化ではなく前提への挑戦が目的で、「それは考えていなかった」という論点を引き出し、決定を記録ファイルに書き戻します。

## 使いどころ

- プランを実装に移す前に、抜け漏れや暗黙の前提を洗い出したいとき
- 「前提を疑って」「計画を深掘り」「プランに穴が無いか」と依頼したいとき
- 設計判断を選択肢と trade-off 付きの質問で1つずつ確定させたいとき

## 設計の要点

- **main agent が直接掘る**: 会話の文脈を持つ main agent が対象を直接読み、前提分析と質問設計を自分で行う。対象が巨大な場合のみ事実調査をサブエージェントに委任し、分析と質問設計は委任しない
- **前提マップ**: 暗黙の前提を6分類（実現可能性 / ユーザー / スコープ / 依存 / 期間 / アーキテクチャ）で洗い出し、「間違っていた場合の被害の大きさ」順に掘る
- **回答から掘る反復**: 1ラウンド2〜3問・上限3ラウンド。回答が露出させた新しい前提を次ラウンドで追う。延長はユーザーに1問だけ確認
- **記録ファイル**: 前提マップ・ラウンドログ・決定は `/tmp/dig/<slug>.md` に集約する。Decisions は設計（OpenSpec の propose や issue-pr-autopilot）への引き継ぎ元になる
- **クロスエージェント**: 構造化質問ツールを抽象化(Claude Code: AskUserQuestion / Codex: request_user_input / 無い環境はテキスト質問)し、環境固有の最適化は SKILL.md 末尾の「環境別ヒント」に分離

## ファイル

- `SKILL.md`: スキル本体。全体方針、記録ファイル形式、前提分析の進め方、質問ラウンドのルール、終了判定、最終サマリー、環境別ヒントを定義します。
- `agents/openai.yaml`: UI メタ情報です。

## リンク

リポジトリ root で以下を実行すると、`~/.claude/skills/dig` と `~/.codex/skills/dig` に symlink を作成できます。

```bash
make link
```
